
 下面这段直接复制粘贴到任何项目的 Claude Code 对话里，它会自动把当前项目情况发布到个人主页。

  ---
  请把当前项目发布到我的个人主页 https://tommickey.cn

  # 操作

  1. 先读懂当前项目：README、package.json/Cargo.toml/pyproject.toml 等，归纳出：
     - 项目名（中文 + 英文）
     - 一句话描述（中文 + 英文，各 ≤80 字）
     - 技术栈（数组）
     - 仓库 URL（如果是 git 仓库，从 `git remote -v` 取）
     - demo URL（如果有部署）
     - 解决的真问题、当前进度、为什么有意思

  2. 在 `/home/tommy/code/myWebsite/src/content/projects/<slug>.md` 创建文件。
     - slug：纯英文小写连字符，简短。如果同名文件已存在，先 Read 看是否是同一项目；
       是同一项目 → 改成更新该文件（更新 `updated` 字段、追加新章节，不要整体重写覆盖用户原有内容）；
       不是 → 换个 slug。

  3. Frontmatter 严格遵循以下 schema（Zod 校验）：

     ---
     title: <中文标题>
     title_en: <英文标题>
     description: <中文一句话>
     description_en: <英文一句话>
     lang: zh
     created: YYYY-MM-DD            # 今天日期；只有新建时才设
     updated: YYYY-MM-DD            # 仅更新已有项目时加
     tags: [a, b, c]                # 全小写英文短词
     stack: [Tech, Tech]             # 技术栈，可中英混排
     repo: https://...               # 可选
     demo: https://...               # 可选
     featured: false                 # 默认 false；除非我让你 featured，否则别动
     draft: false
     ---

  4. 正文 Markdown，结构建议：
     - 引子 1-2 段（讲为什么做、要解决什么真问题，避免营销腔）
     - `## 现在能做什么` 或 `## 现状`（具体能力、当前阶段）
     - `## 技术栈与设计` 或 `## 怎么实现的`（核心思路，不要堆 API 列表）
     - `## 接下来` 或 `## 限制`（诚实说待办和不擅长的）
     - 末尾用 `---` 分隔，加 `## English` 段落，把上面内容译成英文（结构对应即可，不必逐字翻译）

  5. 风格要求：
     - 不要营销词（"赋能/重塑/颠覆/革命性/全新一代"等）
     - 不要 emoji，除非项目本身名字里有
     - 引用其他文章/项目用 `[[标题]]` 双链（仅当确认目标文件存在于
  src/content/{garden,essays,projects}/ 中）
     - 代码示例用 fenced block，标语言

  6. 验证：在 `/home/tommy/code/myWebsite` 跑：
       source ~/.nvm/nvm.sh && nvm use 22.17.1 && pnpm build
     看到 "[build] Complete!" 才算通过。Zod 报错 → 修 frontmatter 重试。

  7. 提交并推送：
       cd /home/tommy/code/myWebsite
       git add src/content/projects/<slug>.md
       git commit -m "project: <slug> - <一句话>"
       git push origin main
     commit message 不要带 Co-Authored-By 行（除非我之前对你说过要带）。

  8. 告诉我：commit hash + 上线 URL（约 60 秒后生效）：
       https://tommickey.cn/projects/<slug>/

  # 边界

  - 只能写 src/content/projects/ 下的这一个文件。不要动其他任何文件、不要改配置、不要改样式。
  - 不要 force push。不要改 main 之外的分支。
  - 当前项目源代码不要复制到个人主页仓库；只描述它。
  - 如果发现敏感信息（密钥、内部 URL、未公开的客户名），从描述里去掉，并提醒我。
  - 拿不准的细节（比如要不要 featured、要不要 demo URL、英文翻译里的某个术语），先问我，别瞎猜。

  开始之前先用一句话告诉我你打算怎么概括这个项目，等我点头再开干。

  ---
  用法说明

  - 每个项目复用一次就行：第一次它会创建
  <slug>.md；以后再用同一段提示词在同一个项目里跑，它会识别出"同一项目"走更新路径。
  - 如果想要 featured（首页精选）：在提示词后加一句 "featured: true"。
  - 如果某个项目内容敏感不想公开：直接告诉那个 Claude Code 不要发布，或者只发布脱敏版本。
  - 想加 essay 而不是 project：把第 2 步路径改成 src/content/essays/，把 frontmatter 里的
  stack/repo/featured 删掉（essays schema 没这几个字段）。
